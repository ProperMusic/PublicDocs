B2B Invoices Version 2

1 Xml file contains 1 transmission.

Below are some simple tables that will explain the structure of the file. To avoid a gigantic, nested mess the file structure has been split into its individual components.

Structure​

Tree Structure​

Lines can either be a "child" of the "message" element or of the order that they belong to.

This option is provided because it can be easier to translate files with a non-nested structure to EDI formats.

Type​

A node in an XML file can be of type "container" (it contains other elements) or "data" (it contains data). 1 Our xml structure attempts to avoid using xml attributes.

Min / Max​

The minimum and maximum number of times this element will appear within this structure. 0/1 means it can appear a minimum of 0 (i.e. the element could be absent) and a maximum of 1 time. 1/99 means it can appear between 1 and 99 times. 1/inf means the element can appear between 1 and "infinite" times. Infinite in the sense that it has no upper bound.

Nested Lines Option​

By default the lines on the invoice are listed separately to the orders, we can, if needed, "nest" the lines within the orders so that each order "contains" its lines.

Default Behaviour

Below is a "toy" example of the default behaviour.

<transmission>
    <header />
    <message>
        <orders>
            <order>
                <orderId>1</orderId>
            </order>
            <order>
                <orderId>2</orderId>
            </order>            
            <lines>
                <line>  
                    <orderId>1</orderId>
                    <code>abc001</code>
                </line>
                <line>  
                    <orderId>1</orderId>
                    <code>abc029</code>
                </line>
                <line>  
                    <orderId>2</orderId>
                    <code>abc088</code>
                </line>
                <line>  
                    <orderId>2</orderId>
                    <code>abc099</code>
                </line>                                                
            </lines>
    </message>

</transmission>
