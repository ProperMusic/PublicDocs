Files & Folders

When you are given the Rich Feed 2.0 connection details, it will connect you to a root folder. Depending on your business relationship with Proper Music, you will then see a maximum of three sub folders:

1. Cost​

This folder contains the cost feeds, if applicable for you, this will contain your Proper Music Dealer price and Net price. Available in XML, JSON & CSV format.

2. Product​

This folder contains the product feeds, with description of each product Available in XML & JSON format.

3. Stock​

This folder contains the stock feed, which shows the stock available to order from Proper Music. The feeds available vary depending on the destination of the feed, Proper Music will configure the feeds as required. Available in XML, JSON & CSV format.

Feeds​

Each Rich Feed will thew following file types:

Full- The weekly full feed of all available products.

Inc-new - Containing records that were not in the last feed update. Inc-update - Containing records that have been changed since the last feed update. NB. This does not include products where only the stock level has changed.

On the product feed there is an additional file Inc-remove - Containing products that have been removed since the last full feed.

Files​

The feeds are split into data chunks, to avoid the files being too large to transfer/read. A standard naming convention is used for all files:

PositionContentExample
StartFeed number
internal ID remains static for a feed		12
2ndfull or inc
Identifies if feed is full or incremental		inc
3rdDate or Date & Time
_Full feed has date in format yymmdd
Incremental has date+time in format yymmddhhmmss		230531150620
4thFile Data chunk
Numeric value 001-999 identifying data chunk		{001}
ExtensionFile Format .json or .xml or .csv.xml
Example:12-inc-230531150620{001} .xml
